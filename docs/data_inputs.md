## Configuration Files

| File | Purpose | Required |
|------|---------|----------|
| `config/policy_params.yaml` | All policy constants: country codes, authority ranges, 232 coverage/rates, floor rates, 301 rates, fentanyl rates, Section 122 expiry, USMCA config, metal content method, stacking method, duty-free treatment, series horizon. Single source of truth for anything that affects the meaning of the tariff series. | Yes |
| `config/revision_dates.csv` | Maps HTS revisions to effective dates and TPC validation dates. Columns: `revision`, `effective_date`, `tpc_date`, `policy_event`. Manually curated when USITC publishes new revisions. | Yes |
| `config/scenarios.yaml` | Counterfactual scenario definitions (baseline, no_ieepa, no_301, no_232, pre_2025). Used by `apply_scenarios.R`. | No (scenarios only) |
| `config/local_paths.yaml` | User-specific paths for external files (import weights, TPC benchmark, Tariff-ETRs repo). Gitignored; template at `local_paths.yaml.example`. | No (weighted/comparison only) |

## What Runs Without What

This matrix shows which outputs are produced based on available inputs. The core build always produces the tariff timeseries; downstream outputs degrade gracefully.

| Scenario | Timeseries | Daily (unweighted) | Daily (weighted) | Weighted ETR | TPC Validation | Quality Report |
|----------|:----------:|:------------------:|:----------------:|:------------:|:--------------:|:--------------:|
| **Core only** (HTS JSON + config + resources) | Yes | Yes | No | No | No | Yes |
| **+ Import weights** | Yes | Yes | Yes | Yes | No | Yes |
| **+ TPC benchmark** | Yes | Yes | No | No | Yes | Yes |
| **+ Import weights + TPC** | Yes | Yes | Yes | Yes | Yes | Yes |
| **Missing HTS JSON** | **Fail** | — | — | — | — | — |
| **Missing policy_params.yaml** | **Fail** | — | — | — | — | — |
| **Missing resource files** | **Fail** | — | — | — | — | — |

**Key behaviors:**
- `00_build_timeseries.R --core-only` forces unweighted daily + quality report, skipping weighted ETR entirely.
- `run_weighted_etr()` checks `config/local_paths.yaml` for `import_weights`; if null or file missing, prints a message and returns without error.
- `plot_etrs()` renders all plots without TPC overlay if TPC data is unavailable.
- TPC validation during build (`07_validate_tpc.R`) is gated on `file.exists(tpc_path)`.
- `run_comparisons.R` reports availability of each data source before running.

## Data Provenance Notes

### Resources derived from Tariff-ETRs

Several committed resource files were originally generated from or validated against the [Tariff-ETRs](https://github.com/Budget-Lab-Yale/Tariff-ETRs) project. They are now maintained as independent repo resources with stable schemas:

| File | Provenance | Status |
|------|-----------|--------|
| `ieepa_exempt_products.csv` | Expanded from Tariff-ETRs US Note 2 product lists + Ch98 + ITA prefixes | Independent; regenerable via `expand_ieepa_exempt.R` |
| `s232_derivative_products.csv` | Originally from Tariff-ETRs config | Independent; manually verified against USITC US Notes |
| `fentanyl_carveout_products.csv` | Originally from Tariff-ETRs | Independent; manually verified against HTS JSON entries |
| `mfn_exemption_shares.csv` | From Tariff-ETRs Census calculated duty data | Independent; regenerable from Census data |
| `metal_content_shares_bea_hs10.csv` | Generated by Tariff-ETRs `build_metal_content_shares.R` from BEA I-O | Could be regenerated from BEA source data |

None of these require a live connection to the Tariff-ETRs repo. The core build uses only repo-owned resources, official USITC downloads, and explicit policy configuration.
